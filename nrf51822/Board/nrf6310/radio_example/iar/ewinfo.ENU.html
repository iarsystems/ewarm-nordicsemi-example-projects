Example description

  
Example project on transmitting and receiving data using the NRF_RADIO peripheral

DESCRIPTION
===========
 
   The transmitter is configured to continuously send packets. The first payload
  byte of the packet is take from GPIO Port 0 which can connected to, for
  example, the buttons on an nRFgo board. After transmission, the first
  payload byte of the packet is sent to P1 which can be used, for example, to
  light up the LEDs on an nRFgo board.

   The receiver is configured to continuously receive packets from the corresponding
  transmitter project. The first payload byte of the packet is sent to P1 which can
  be used, for example, to light up the LEDs on an nRFgo board.
 
  For more detailed information see nRF51 SDK.

COMPATIBILITY
=============

   The example project is compatible with, on Nordic Semi nRF6310 - PCA10005 evaluation
  boards. By default, the project is configured to use the J-Link SWD interface.

CONFIGURATION
=============

   Make sure that the following jumpers and swiches are correctly configured on the
  nRF6310 evaluation board:
  Switches:
  S8  - VBUS
  S9  - ON
  S11 - ON
  
  PCA10005 board:
  Jumpers:
   K1 - 2-3
   P4 - 1-3 2-4 (32 kHz crystal)

  The Blinky application is downloaded to the iFlash.

GETTING STARTED
===============

  1) Build and download the program.

  2) Run the program.
