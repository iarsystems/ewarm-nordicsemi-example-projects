Detailed Description

This example needs that the slave is configured to transmit the received bytes. That is the slave behaves as a loopback device for the master. The loopback can also be achieved without using a slave device at all by wiring MOSI and MISO pins of the spi master together. TX_RX_MSG_LENGTH number of bytes are transmitted through the master and the received bytes are verified to be the same as transmitted. IF there is an error, gpio pin for relevant spi module is set high to show the error

See also:
ERROR_PIN_SPI0,
ERROR_PIN_SPI1. If there is error from both modules that is if both pins are set high, then this application loops for ever
