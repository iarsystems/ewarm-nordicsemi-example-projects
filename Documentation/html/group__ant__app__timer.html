Detailed Description

It enables the application to create multiple timer instances based on the RTC1 timer peripheral. Applications using app_timer must have a configuration file named app_timer_cfg.h (see ble_sdk_apps_config_files for details).

Note:
Due to a problem in the way the application uses the RTC peripheral, timer interrupts may be lost. Therefore, the current implementation is not reliable, and should not be used in production code.

Parameters:
[in]issue_timeout_evt_funcFunction for letting the application control how or when timeout handlers are executed. Normally this will be used for making the timer module pass timeout events to the scheduler (by supplying a pointer to app_sched_timer_event_schedule()). Set to NULL to make the timer module call the timeout handler directly from the timer interrupt handler.
