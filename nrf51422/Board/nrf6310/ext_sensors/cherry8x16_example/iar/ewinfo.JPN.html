Example description

  
Cherry 8x16 keyboard example

DESCRIPTION
===========
 
   This example will detect the characters by scanning the matrix for any state change (key(s) press or release).
  If any changes from the previous state is identified, then the received key codes are converted from hid to
  ascii which are transmitted over uart and can be listened by any COM listener like example:PUTTY (this is
  default behavior). However commenting out USE_UART can be done to to output modifier key presses to LEDs
  through GPIOs.
  
   This example is meant to be used with the nRF6423 board. Connect nRF6310 P1 to LEDs.
  Connections between the two boards:
  - nRF6423 - nRF6310
  - (column lower)  P3   -   P0
  - (column higher) P4   -   P1
  - (rows)          P2   -   P2
  
   Can also be used with other keyboards, in which case row and column mapping have to be changed in this example
 
  For more detailed information see nRF51 SDK.

COMPATIBILITY
=============

   The example project is compatible with, on Nordic Semi nRF6310 - PCA10006 evaluation
  boards. By default, the project is configured to use the J-Link SWD interface.

CONFIGURATION
=============

   Make sure that the following jumpers and swiches are correctly configured on the
  nRF6310 evaluation board:
  Switches:
  S8  - VBUS
  S9  - ON
  S11 - ON
  
  PCA10006 board:
  Jumpers:
   K1 - 2-3
   P4 - 1-3 2-4 (32 kHz crystal)

  The Blinky application is downloaded to the iFlash.

GETTING STARTED
===============

  1) Build and download the program.

  2) Run the program.
